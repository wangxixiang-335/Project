# 项目文件结构说明

## 📁 项目文件夹结构

### 🏗️ 核心文件
- `package.json` - 项目依赖和脚本配置
- `package-lock.json` - 锁定依赖版本
- `.env` / `.env.example` - 环境变量配置
- `.gitignore` - Git忽略文件配置

### 📂 文件夹说明

#### 🗄️ `database/`
数据库相关文件
- `.sql` 文件 - 数据库表结构、迁移脚本
- `*database*.js` - 数据库初始化脚本
- `*table*.js` - 数据表相关脚本

#### 🔍 `debug/`
调试和检查脚本
- `debug*.js` - 调试脚本
- `check*.js` - 数据检查脚本
- `diagnose*.js` - 诊断脚本
- `simple*.js` - 简化调试脚本

#### 📚 `docs/`
项目文档
- `.md` 文件 - 各种说明文档、指南
- `.pdf` 文件 - 项目设计文档
- API文档、修复指南、部署指南等

#### 🔧 `fixes/`
修复和初始化脚本
- `fix*.js` - 修复脚本
- `create*.js` - 创建脚本
- `*auth*.js` - 认证相关脚本

#### 🧪 `tests/`
测试文件
- `test*.js` - 测试脚本
- `*test*.js` - 各种测试文件
- 验证和检查脚本

#### 🚀 `startup-scripts/`
启动和配置脚本
- `start*.js` - 启动脚本
- `.bat` 文件 - 批处理启动脚本
- `postman_collection.json` - API测试集合

#### 🌐 `temp-html/`
临时HTML文件
- 测试页面
- 调试页面
- 临时前端文件

#### 📁 核心项目文件夹
- `src/` - 后端源代码
- `scripts/` - 项目脚本
- `supabase/` - Supabase配置
- `temp-frontend/` - 前端临时文件

---

## 🎯 使用建议

### 开发时
1. 主要代码在 `src/` 和 `temp-frontend/` 中
2. 数据库操作参考 `database/` 中的脚本
3. 调试问题使用 `debug/` 中的工具

### 测试时
1. 运行 `tests/` 中的测试脚本
2. 使用 `temp-html/` 中的页面进行前端测试

### 部署时
1. 参考 `docs/` 中的文档
2. 使用 `startup-scripts/` 中的启动脚本
3. 根据需要运行 `fixes/` 中的修复脚本
